# CODE_OF_CONDUCT.md

# Código de Conduta

Bem-vindo ao projeto **Conta Comigo**.

Nosso objetivo é construir um ambiente colaborativo, respeitoso e profissional, onde todas as pessoas possam contribuir de forma segura, produtiva e transparente.

---

# Nosso Compromisso

Todos os participantes do projeto comprometem-se a:

* tratar todas as pessoas com respeito;
* agir com profissionalismo;
* colaborar de forma construtiva;
* aceitar críticas técnicas de maneira respeitosa;
* compartilhar conhecimento;
* priorizar o sucesso coletivo do projeto.

---

# Comportamentos Esperados

São considerados comportamentos positivos:

* comunicação educada e objetiva;
* disposição para ajudar outros colaboradores;
* abertura para aprender e ensinar;
* foco na resolução de problemas;
* respeito às diferentes opiniões técnicas;
* documentação clara das decisões;
* revisão de código baseada em critérios técnicos.

---

# Comportamentos Inaceitáveis

Não serão tolerados:

* ataques pessoais;
* assédio ou intimidação;
* linguagem ofensiva ou discriminatória;
* exposição de informações confidenciais;
* sabotagem do trabalho de outros colaboradores;
* discussões improdutivas ou desrespeitosas;
* envio deliberado de código malicioso.

---

# Comunicação

As discussões devem permanecer focadas no problema técnico.

Ao discordar de uma proposta:

* apresente argumentos técnicos;
* proponha alternativas;
* mantenha uma postura respeitosa;
* evite críticas pessoais.

Discordâncias fazem parte do desenvolvimento de software e devem contribuir para a melhoria do projeto.

---

# Revisão de Código

Durante o Code Review:

* critique o código, nunca a pessoa;
* explique o motivo das sugestões;
* utilize referências técnicas sempre que possível;
* reconheça boas soluções quando apropriado.

O objetivo da revisão é aumentar a qualidade do software e compartilhar conhecimento.

---

# Decisões Técnicas

As decisões arquiteturais devem ser baseadas em:

* requisitos do projeto;
* desempenho;
* segurança;
* manutenção;
* simplicidade;
* escalabilidade.

Preferências pessoais não devem prevalecer sobre critérios técnicos.

---

# Uso de Inteligência Artificial

Ferramentas de IA podem ser utilizadas como apoio ao desenvolvimento.

Entretanto:

* todo código deve ser revisado antes do envio;
* a autoria da contribuição implica responsabilidade pelo resultado;
* a documentação oficial do projeto prevalece sobre qualquer sugestão automática.

---

# Privacidade

Todos os colaboradores devem proteger:

* dados dos usuários;
* credenciais;
* informações financeiras;
* segredos comerciais;
* chaves de acesso;
* informações internas do projeto.

Nunca compartilhe informações confidenciais em canais públicos.

---

# Resolução de Conflitos

Em caso de divergências:

1. tente resolver diretamente com diálogo respeitoso;
2. envolva um revisor técnico quando necessário;
3. encaminhe questões arquiteturais ao Tech Lead;
4. registre decisões importantes na documentação oficial.

---

# Aplicação

Este Código de Conduta aplica-se a:

* código-fonte;
* documentação;
* Issues;
* Pull Requests;
* discussões;
* reuniões técnicas;
* canais oficiais do projeto.

---

# Consequências

O descumprimento deste documento poderá resultar em:

* solicitação de ajustes na conduta;
* rejeição de contribuições;
* restrição temporária de participação;
* remoção permanente do projeto, quando necessário.

As medidas adotadas serão proporcionais à gravidade da situação.

---

# Princípios

Este projeto acredita que excelentes sistemas são construídos por equipes que valorizam:

* respeito;
* responsabilidade;
* transparência;
* aprendizado contínuo;
* excelência técnica;
* colaboração.

Nossa prioridade é manter um ambiente onde todos possam contribuir com segurança, confiança e profissionalismo.
