# Versionamento

O projeto utiliza:

Semantic Versioning

```
MAJOR.MINOR.PATCH
```

Exemplo:

```
1.4.2
```

Onde:

* MAJOR: mudanças incompatíveis.
* MINOR: novas funcionalidades compatíveis.
* PATCH: correções e melhorias sem quebra de compatibilidade.
