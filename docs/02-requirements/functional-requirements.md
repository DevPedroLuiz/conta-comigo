# Functional Requirements — Conta Comigo

## Objetivo

Este documento define todas as funcionalidades que o sistema deve implementar.

---

## Autenticação

- FR-001: O sistema deve permitir cadastro de usuários.
- FR-002: O sistema deve permitir login com e-mail e senha.
- FR-003: O sistema deve suportar refresh token.
- FR-004: O sistema deve permitir logout.
- FR-005: O sistema deve permitir recuperação de senha.

---

## Usuários

- FR-006: O sistema deve permitir atualização de perfil.
- FR-007: O sistema deve permitir múltiplas contas financeiras por usuário.

---

## Contas

- FR-008: O sistema deve permitir criação de contas financeiras.
- FR-009: O sistema deve permitir edição de contas.
- FR-010: O sistema deve permitir listagem de contas.
- FR-011: O sistema deve calcular saldo automaticamente com base nas transações.

---

## Transações

- FR-012: O sistema deve permitir criação de transações.
- FR-013: O sistema deve permitir edição de transações.
- FR-014: O sistema deve permitir exclusão lógica de transações.
- FR-015: O sistema deve categorizar transações.
- FR-016: O sistema deve suportar receitas e despesas.

---

## Categorias

- FR-017: O sistema deve permitir criação de categorias personalizadas.
- FR-018: O sistema deve fornecer categorias padrão.

---

## Transferências

- FR-019: O sistema deve permitir transferências entre contas.
- FR-020: O sistema deve garantir consistência entre débito e crédito.

---

## Cartões

- FR-021: O sistema deve permitir cadastro de cartões.
- FR-022: O sistema deve permitir controle de limite.
- FR-023: O sistema deve permitir parcelamento de compras.

---

## Fluxo de Caixa

- FR-024: O sistema deve gerar fluxo de caixa automaticamente.
- FR-025: O sistema deve permitir filtros por período.

---

## Relatórios

- FR-026: O sistema deve gerar relatórios financeiros.
- FR-027: O sistema deve exportar relatórios em PDF e Excel.

---

## IA Financeira

- FR-028: O sistema deve fornecer sugestões financeiras.
- FR-029: O sistema deve classificar transações automaticamente.
- FR-030: O sistema deve exigir confirmação do usuário para qualquer ação financeira.

---

## Notificações

- FR-031: O sistema deve enviar notificações de eventos financeiros.
- FR-032: O sistema deve suportar notificações por e-mail e push.

---

## Auditoria

- FR-033: O sistema deve registrar ações críticas.
- FR-034: O sistema deve manter logs imutáveis.
